Technical Assignment Round 2:
Automated Book Cover Validation System
Project Overview
BookLeaf Publishing processes 100-150 book covers monthly for their Bestseller Breakthrough
Package. Your task is to build an automated system to detect and report book cover layout
issues, reducing manual review time by 80% while maintaining 90%+ accuracy.
The most critical issue to detect is author names overlapping with the "21st Century Emily
Dickinson Award" badge placement area.
Core Technical Requirements
1. Computer Vision Detection System
Critical Detection Capabilities (95% accuracy required):
● Text overlap with award badge area
● Author name positioning conflicts within safe zones
Additional Detection Requirements:
● Text-to-border spacing violations
● Back cover text alignment issues
● Image resolution and quality assessment
2. Cover Specifications & Rules
Dimensions:
● Front cover: 5x8 inches
● Safe area margins: 3mm on each side, 9mm from bottom of front cover
● Bottom 9mm reserved for "Winner of 21st Century Emily Dickinson Award" emblem
Author Name Placement:
● Can be positioned anywhere on front cover
● Must stay within designated safe area
● Cannot overlap with award badge zone
Refer to the Youtube video below to understand our safe area margins and text placement
correctly (Timestamp 4:20): https://www.youtube.com/watch?v=Z9wxMeo624k
Quality Standards:
● Cover must appear clear with no pixelation
● Text must be legible and properly aligned
● All elements must respect safe margins
3. Workflow Automation Requirements
Trigger Mechanism:
● Book covers uploaded to designated Google Drive folder
● File naming convention: ISBN_text (e.g., 1234567890123_text.pdf)
● Supports PDF and PNG formats

4. Status Classifications
